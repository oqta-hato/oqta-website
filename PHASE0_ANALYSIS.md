# Phase 0 解析：現状マップ × ブランド・レンズ仕分け（2026-05-29）

> 対象: Figma `OQTA-Renewal`(AQ2NpCInRanyjeU3CXGDJ5)。判断レンズ: [[BRAND_FOUNDATION.md]]。
> 素材: ローカル保有（`public/assets/`）。デザイントークン: `primary #003da6` / accent `#e5ecf6`（変数はほぼ未使用）。

## A. 構造マップ
- **サービスサイト(24:2, 1440×10619)**: Hero → About(長文) → サービス → 使い方 → OQTAに込めた想い → 利用者の声 → 実績(受賞) → オファー → FAQ → footer
- **LP(34:582, 1440×6967)**: Hero(レビュー前面) → ABOUT → **こんな方々にお使いいただいています** → 実績 → 利用者の声 → サービス → FAQ → フッター
- 共通(再利用可): Hero / 使い方 / 実績 / 利用者の声 / サービス / FAQ

## B. デザインシステム現状
- 色: `#003da6`(紺) / `#e5ecf6`(accent)。**LPだけCTAがピンク**＝トンマナ不一致(要統一)。
- フォント(現実装): Zen Kaku / Italianno / Inter。
- 空気: ソフト・余白あり・ライフスタイル写真＝**新ブランドの素地はある**。ただし「商品を購入する」＝購入/製品寄りの重心。

## C. ブランド・レンズ仕分け（KEEP / TRANSFORM / CUT / ADD）

**KEEP**: 余白のクリーンな世界観 / 実話・利用者の声・レビュー(語らずストーリーの核) / 使い方STEP線画 / 製品の地域名運用(Heart Poppo・INORI Clock)。

**TRANSFORM**:
- Hero「実家にハトを贈ろう」(購入導線寄り) → INORI/想いを送る の文化メッセージ＋英語主役(pray不使用)。「実家に〜」はCTAに降格。
- CTA「商品を購入する」中心 → 購入と“応援(神社モデル/お賽銭)”を並置・pink/navy統一。
- 利用者の声 → ZINEの実話・LINE証言・3つの利用シーン(介護施設/仏壇/分身時計)に格上げ＝“定義できないことの証明”。
- 全体を日英バイリンガル(英語主役級)に。

**CUT/降格**: 製品スペック的・家電販売的な見せ方の比重を下げる / レガシー表記(OQTA HATO 等、廃止済)。

**ADD（ZINEにあるがデザインに無い＝リブランドの本体）**:
- 思想: 言葉を使いすぎた時代 / 余白という思想 / 見返りを求めない祈り
- **inoree.world 共鳴マップ**(ライブ・カウンター)＝インタラクティブの目玉
- **京都の鳩を鳴らすQR体験**
- 創業ストーリー＋僧侶ファウンダー＋**神社モデル(応援＝外部決済)**
- Hub(YouTube / note / Heart Poppo Radio)
- 英語コピー全般(INORI定義)

## D. 重要な発見
1. **LPには既に「こんな方々に」＝クラスター別ペルソナが実装済み**: ①遠方の親 ②感謝を伝えたい ③**介護施設・長期入院** ④**海外赴任・留学**。→ JQのLP構想(クラスター別1枚)と完全一致・流用可能。
2. デザインの素地(余白・ソフト)は使える。**本質は「購入・製品」→「文化・想い・応援」への重心移動**。
3. 最大のギャップ＝ZINEの深い思想・inoree.world・京都体験・創業/僧侶/神社モデル・英語 が現デザインに不在。**ここが“足す”領域**。

## 結論
現デザインは**素体としては使える**が、リブランドの本体は「**ADD領域を載せ、重心を購入→文化へ移し、サービスサイトをZINEのWeb版に再構成する**」こと。
